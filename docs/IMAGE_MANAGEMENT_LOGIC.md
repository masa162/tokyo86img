# 画像・アセット運用ロジック定義書（ユーザー思想と設計指針）

## 1. 運用の背景と哲学
本システムにおける画像管理は、単なるデータの保存ではなく、**「クリエイターの運用負荷を極限まで減らすこと」** と **「コンテンツのセキュリティ（予測不能性）」** の両立を目的とする。

### A. 「植物観察」モデルからの着想（連番URLの利便性）
- **文脈**: 数十〜百枚単位の写真を扱う際、1枚ずつURLを扱うのは不可能。
- **解決策**: 1つの「箱（バッチ）」にアップロードされた画像は、`img.unbelong.xyz/{batch-id}/001.webp` から始まる完全な連番で配信される。
- **メリット**: Markdown出力ボタン一発で、100枚の画像を即座にブログやエピソードに貼り付け可能にする。

### B. ランダム英数字IDによる「予測不能な箱」の設計
- **文脈**: 完全な連番（000001, 000002...）だと、URLを書き換えることで未公開の前後画像にアクセスできてしまうリスクがある。
- **解決策**:
    - 各「箱」には **ランダム英数字 6桁** の ID を自動発番する。
    - URL例: `img.unbelong.xyz/6kH2p9/001.webp`
- **メリット**: 箱の名前がわからない限り外部からは推測不能。一方で箱の中身は連番であるため、クリエイターによる編集（Markdown上での番号調整）も容易。

## 2. 実装コンテキストの分類

### A. 単品画像（イラスト運用）
- **対象**: `unbelongillust`
- **挙動**: アップロード時に自動で英数字 ID が発番される（ユーザー入力の手間を省く）。
- **特徴**: 1投稿1画像の基本スタイル。

### B. 複数画像（コミック/Webtoon運用）
- **対象**: `unbelongcomic`
- **挙動**: 複数画像をドラッグ＆ドロップで一括アップロード。1つの「箱（バッチID）」に紐付け、自動で 001, 002... と連番を振る。
- **出力**: アップロード完了後、その箱の全画像の連番 Markdown（または URL リスト）をテキストで一括出力。

## 3. 技術的要件：Cloudflare Images の活用
- **デバイス最適化**: 横幅 2000px などの大きなオリジナル画像を用意。
- **配信ロジック**: スマホ・PC・TVなど、閲覧環境に合わせて Cloudflare Images のリサイズ・最適化をバックエンドで動的に適用する。

## 4. 運用フローのゴール
1. 管理画面で画像をドロップ。
2. 自動で「ランダム6桁ID/連番.webp」のセットが生成される。
3. 生成されたMarkdownをコピーして、エピソードの本文に貼り付けて完成。
