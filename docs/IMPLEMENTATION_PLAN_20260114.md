# 実装プラン: Webtoon 展開の強化 (2026/01/14)

## 目的
「箱（バッチ）」アップロード機能を完全に Webtoon 配信フローへ統合し、作品管理から公開までをシームレスに行えるようにする。

## 1. バックエンド API 強化 (`unbelong/apps/api`)
- [ ] **Works CRUD エンドポイントの完備**
    - `POST /api/works`: 新規作品作成
    - `PUT /api/works/:id`: 作品情報更新
    - `DELETE /api/works/:id`: 作品削除 (関連するエピソードへの影響も考慮)
- [ ] **Episodes CRUD の安定化**
    - 既存の POST/PUT/DELETE が正しく動作するか再検証

## 2. 管理画面の機能拡張 (`unbelong/apps/admin`)
- [ ] **作品管理 (`/works`) の完全実装**
    - [ ] `Works.tsx`: 一覧からの削除機能追加、編集画面への遷移。
    - [ ] `WorkForm.tsx` (新規): 作品のタイトル、スラッグ、タイプ（マンガ/イラスト）、説明、OGP 等を設定できるフォーム。
    - [ ] `WorkNew.tsx` / `WorkEdit.tsx` (新規): 作品作成・編集ページ。
- [ ] **エピソードフォームの調整**
    - [ ] `EpisodeForm.tsx`: バッチで生成された Markdown をそのまま貼り付けやすく調整。プレビュー機能の検討。

## 3. Webtoon フロントエンドの実装 (`unbelongcomic`)
- [ ] **API 連携**
    - セントラル API (`unbelong-api`) から作品一覧・エピソード一覧・本文 (Markdown) を取得するようにする。
- [ ] **ビューアーの磨き込み**
    - `img.unbelong.xyz` 経由の最適化済み画像を、隙間なく連続表示する UI の完成。
    - 次の話・前の話へのナビゲーション。

## 4. 運用フローの完成イメージ
1. **箱作成**: `Batch Upload` で Webtoon 原稿を一括アップロード。
2. **MDコピー**: 生成された `![](...)` の Markdown リストをコピー。
3. **エピソード作成**: `作品・マンガ管理` -> 各作品の `エピソード追加` でコピーした MD をペースト。
4. **公開**: 公開ステータスにすると `comic.unbelong.xyz` に即座に反映。

## 完了定義
- 管理画面で作品の新規作成・編集・削除ができること。
- アップロードした画像セットが、コミックフロントエンドで Webtoon として正しく閲覧できること。
